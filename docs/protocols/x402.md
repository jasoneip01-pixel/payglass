# x402 协议集成指南

## 协议概述

x402 是基于 HTTP 402 Payment Required 状态码的 Agent 支付协议。核心创新：支付嵌入 HTTP 请求本身，不需要账户、会话或钱包弹窗。

### 标准流程

```
Client Agent                    Server (Resource)              Circle Gateway
     │                                │                              │
     │──GET /api/v1/chat─────────────>│                              │
     │<──402 Payment Required─────────│                              │
     │   X-Payment: x402              │                              │
     │   amount=0.01 USDC             │                              │
     │                                │                              │
     │──EIP-3009 sign(amount, nonce)  │                              │
     │                                │                              │
     │──GET /api/v1/chat─────────────>│                              │
     │   X-Payment: x402              │                              │
     │   Authorization: Bearer <sig>  │                              │
     │                                │──verify(sig)────────────────>│
     │                                │<──✅ verified────────────────│
     │<──200 OK + response data───────│                              │
     │                                │                              │
     │                                │──batch_settle───────────────>│
     │                                │<──tx_confirmed───────────────│
```

## PayGlass 中的 x402 Observer

### 初始化

```python
from engine.tracer import PayGlassTracer
from engine.protocols.x402 import X402Observer

tracer = PayGlassTracer()
x402 = X402Observer(tracer)
```

### 追踪一笔支付

```python
trace_id = x402.trace_payment(
    resource_url="https://api.openai.com/v1/chat/completions",
    amount_usdc=0.01,
    source_agent="research-agent-42",
    chain="arbitrum",
)

# 查看完整追踪
trace = tracer.get_trace(trace_id)
for event in trace.events:
    print(f"[{event.event_type}] {event.status.value}")
```

### 输出示例

```
[402_response] pending
[signature] pending
[payment_header] pending
[verification] verified
[resource_delivered] verified
[batch] batched
[settlement] settled
```

### 7 个事件详解

| # | 事件类型 | 状态 | 说明 |
|:--:|------|:--:|------|
| 1 | `402_response` | PENDING | 服务端返回 402，要求支付 |
| 2 | `signature` | PENDING | Agent 生成 EIP-3009 授权签名 |
| 3 | `payment_header` | PENDING | Agent 重试请求，带上 Payment 头 |
| 4 | `verification` | VERIFIED | Gateway 验证签名通过 |
| 5 | `resource_delivered` | VERIFIED | 服务端返回资源 |
| 6 | `batch` | BATCHED | 支付被纳入批量结算批次 |
| 7 | `settlement` | SETTLED | 链上结算完成 |

### 批量结算

```python
# 多笔支付合并一批
x402.settle_batch(
    trace_id="abc123",
    batch_id="batch-x402-20260605-001",
    tx_hash="0xabcdef...",
    chain="arbitrum",
)
```

## 调试常见问题

### 问题 1：402 响应未触发

**症状**：Agent 的 HTTP 请求直接返回 200，没有触发支付流程。

**排查**：
```python
trace = tracer.get_trace(trace_id)
if not any(e.event_type == "402_response" for e in trace.events):
    print("❌ Service didn't return 402 — check if endpoint supports x402")
```

**根因**：服务端未配置 x402 中间件，或资源 URL 需要 HTTPS。

### 问题 2：签名验证失败

**症状**：verification 事件状态为 FAILED。

**排查**：
```python
for e in trace.events:
    if e.event_type == "verification" and e.status == EventStatus.FAILED:
        print(f"Signature verification failed: {e.error}")
        print(f"Metadata: {e.metadata}")
```

**常见原因**：
- EIP-3009 签名格式错误（nonce 重复、过期）
- Gateway 钱包余额不足
- Token 合约地址不匹配

### 问题 3：批量结算延迟

**症状**：支付 VERIFIED 但长时间未 SETTLED。

**排查**：
```python
if trace.is_complete:
    print(f"Settlement took {trace.settlement_time_ms}ms")
else:
    print("Payment not yet settled — check batch interval")
```

**说明**：批量结算是周期性的（通常 30 秒 - 5 分钟），不是实时的。这是设计行为，不是 bug。

## 与 Nanopayments 的区别

| | x402 | Nanopayments |
|------|------|------|
| **入口** | HTTP 402 状态码 | 直接 EIP-3009 授权 |
| **协议层** | HTTP 原生 | Circle Gateway API |
| **目标用户** | API/Web 服务商 | 任意 Agent 间支付 |
| **最小金额** | 无限制 | $0.000001 |
| **标准化** | x402 社区标准 | Circle 产品 |

两种协议可以通过 Circle Gateway 统一结算——x402 支付也用 Circle Gateway 做批量结算。这是 PayGlass 跨协议观测的核心价值：同一笔 Gateway 批量交易可能同时包含 x402 和 Nanopayments。
